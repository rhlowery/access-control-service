# Access Control Service (ACS) - Project Site

Welcome to the official documentation for the **Access Control Service (ACS)**.

ACS is a centralized security and governance middleware designed to manage access policies across heterogeneous data catalogs and identity providers.

## Key Features

- **Multi-Catalog Governance**: Unified access control for Databricks Unity Catalog, Hive Metastore, and more.
- **Persona-Based RBAC**: Dynamic persona resolution (ADMIN, APPROVER, REQUESTER) based on identity provider claims and local configuration.
- **Modern Architecture**: Built on Quarkus (Java) and React (TypeScript) for high performance and scalability.
- **Cloud Native**: Managed via Helm with integrated observability and security best practices.
